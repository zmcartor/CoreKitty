CoreKitty

"Cat"egory for Core Data 'findBy' accessors

Why ?

Writing Core Data lookup boilerplate is a bummer. The findBy methods from other dynamic languages make data lookup easy. Other frameworks exist for smoothing over Core Data, however I wanted something easy and lightweight.

Also Metaprogramming in ObjC is fun!

Installation

  • Add the NSManagedObject+CoreKitty category to your project.
  • #import on any NSManagedObject subclass
  • Because of the nature of ARC, selectors you'd like to use need to be declared in either the .h or a private class extension. It is not necessary to declare a findBy selector for every single field in the model.

Example:

// MyModel.h
+ (NSArray *)findByFirstName:(NSString *);
+ (NSArray *)findBySomeFieldName:(NSString *);

That's it! Now running :

NSArray *results = [MyModel findByName:@"Homer"];

Will fetch all people models where name = Homer, autometa-magically!
